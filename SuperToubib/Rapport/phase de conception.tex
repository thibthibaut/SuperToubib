\section*{Introduction}

Le but de ce projet est de réaliser une base de données visant à être utilisée dans un milieu hospitalier. Pour ce faire, nous devons utiliser simultanément nos connaissances acquises au début de ce semestre lors du cours sur les bases de données et nos connaissances sur notre cours actuel: le java. Nous exploiterons une base de données déjà crée et stockée sous MySql à travers une application Java codée sous NetBeans proposant une interface graphique agréable et ergonomique pour l'utilisateur.
Nous avons donc dû passer par une phase de conception afin de vraiment comprendre et anticiper tout les enjeux de ce vaste projet. Pour se faire, nous avons suivi la démarche proposée dans le sujet, à savoir:\\
1/ Établissement du diagramme de classes. \\
2/ Répartition des classes dans les différents modules. \\
3/ Répartition des méthodes dans les différents modules, en précisant les entrées et les sorties. \\
4/ Schémas de l'interface graphique. \\
5/ Les algorithmes ou organigrammes des différentes méthodes. \\
Nous vous présenterons dans cette phase de conception ces différents points, en essayant d'expliquer de la manière la plus claire possible nos choix.

Nous avons décidé d'organiser notre projet en quatre packages: un package Modèle,  un package Vu, un package Contrôleur et un package Connexion, qui a pour but de relier l'application Java à la base de données. Le choix de ces packages s'est fait dans un soucis d'optimisation et d'indépendance des modules afin de pouvoir par la suite se répartir au mieux les tâches. De plus, respecter la notion de programmation orienté objet à été un point central de notre réflexion. En effet, nous avons essayé de penser les classes de la manière la plus logique possible afin qu'elles représentent au mieux les différents objets que nous manipulons.

\section{Répartition des méthodes dans les différents modules}

\subsection*{Dans le module Recherche :}
\emph{Les entrées sorties vont dépendre de l'objet qui implémente l'interface. Les méthodes ci-dessous seront polymorphes et définies pour chaque objet du modèle.}\\
Rechercher un patient (entrée$\rightarrow$champ de recherche/ Sortie$\rightarrow$Liste d'Array d'infos)\\
Rechercher un docteur(entrée$\rightarrow$champ de recherche/ Sortie$\rightarrow$Liste d'Array d'infos)\\
Rechercher un(e) infirmier(ère) (entrée$\rightarrow$champ de recherche/ Sortie$\rightarrow$Liste d'Array d'infos)\\
Rechercher un hospitalisation (entrée$\rightarrow$champ de recherche/ Sortie$\rightarrow$Liste d'Array d'infos)\\
Recherche de lit par service (entrée$\rightarrow$nom du service, Sortie$\rightarrow$Numéro du lit disponible ou une exception levée si pas de lits disponibles.)\\

\subsection*{Dans le module Mise à jour}
\emph{Les entrées sorties vont dépendre de l'objet qui implémente l'interface. Les méthodes ci-dessous seront polymorphes et définies pour chaque objet du modèle.}\\
Modifier l'attribut d'une personne(entrée$\rightarrow$L'attribut(nom,prénom,adresse...) , pas de sortie)\\
Modifier l'attribut d'un patient (entrée$\rightarrow$L'attribut, pas de sortie)\\
Modifier l'attribut d'un médecin(entrée$\rightarrow$L'attribut, pas de sortie)\\
Modifier l'attribut d'un infirmier(entrée$\rightarrow$ L'attribut, pas de sortie)\\
Modifier le nom d'un service(entrée$\rightarrow$String, pas de sortie)\\
Modifier le surveillant d'une chambre(entrées$\rightarrow$Chambre et surveillant, pas de sortie)\\
\subsection*{Dans le module Génération et historisation des activités}

\noindent Ajouter un patient\\
Supprimer un patient (Le mettre dans l'historique)\\
Ajouter un médecin\\
Ajouter un(e) infirmier(ère)\\
Ajouter une hospitalisation\\
Supprimer une hospitalisation(archive)\\
Nouvelles données(champs) dans tables existante\\
Nouvelles tables : CREATE TABLE, DROP TABLE\\

\subsection*{Dans le module Connexion }
\noindent Se connecter à la BDD (entrées$\rightarrow$IP, pseudo, mot de passe..., pas de sortie)\\
Exécuter une requête SQL sur la BDD(entrée$\rightarrow$La requête(String), sortie$\rightarrow$le résultat de la requète)\\
